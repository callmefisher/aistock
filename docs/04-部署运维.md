## 部署要求

### 系统要求

| 项目 | 最低要求 | 推荐配置 |
|------|---------|---------|
| CPU | 2核 | 4核+ |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 20GB | 50GB+ |
| 操作系统 | macOS 10.15+, Windows 10+, Ubuntu 18.04+ | 最新稳定版 |

### 软件要求

- **Docker**: 20.10+ 
- **Docker Compose**: 2.0+ (或 Docker Compose V1 1.29+)
- **Git**: 2.20+

### 支持的平台

| 操作系统 | 架构 | 状态 | 备注 |
|---------|------|------|------|
| macOS | Intel (x86_64) | ✅ 完全支持 | |

## 快速部署

### 一键部署（推荐）

```bash
# 克隆项目
git clone git@github.com:callmefisher/aistock.git
cd aistock

# 执行部署脚本
chmod +x deploy.sh
./deploy.sh
```

部署脚本会自动：
1. ✅ 检测操作系统和架构
2. ✅ 检查Docker环境
3. ✅ 检测端口占用并自动切换
4. ✅ 拉取基础镜像（仅首次）
5. ✅ 构建自定义镜像
6. ✅ 启动所有服务
7. ✅ 显示访问地址

### 首次部署流程

```bash
# 1. 配置环境变量
cp .env.example .env
nano .env  # 或使用其他编辑器

# 2. 执行部署
./deploy.sh

# 3. 查看服务状态
docker compose ps

# 4. 查看日志
docker compose logs -f backend
```

## 备份和恢复

### 数据备份

```bash
# 备份MySQL数据
docker compose exec mysql mysqldump -u root -p stock_pool > backup_$(date +%Y%m%d).sql

# 备份Redis数据
docker compose exec redis redis-cli BGSAVE
docker cp stock_redis:/data/dump.rdb ./backup/redis_$(date +%Y%m%d).rdb

# 备份上传的文件
tar -czf data_backup_$(date +%Y%m%d).tar.gz data/
```

### 数据恢复

```bash
# 恢复MySQL数据
cat backup_20240101.sql | docker compose exec -T mysql mysql -u root -p stock_pool

# 恢复Redis数据
docker cp ./backup/redis_20240101.rdb stock_redis:/data/dump.rdb
docker compose restart redis

# 恢复文件数据
tar -xzf data_backup_20240101.tar.gz
```

### 自动备份脚本

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="./backups"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份MySQL
docker compose exec -T mysql mysqldump -u root -p${MYSQL_ROOT_PASSWORD} stock_pool > $BACKUP_DIR/mysql_$DATE.sql

# 备份Redis
docker compose exec redis redis-cli BGSAVE
sleep 5
docker cp stock_redis:/data/dump.rdb $BACKUP_DIR/redis_$DATE.rdb

# 备份文件
tar -czf $BACKUP_DIR/data_$DATE.tar.gz data/

# 删除7天前的备份
find $BACKUP_DIR -type f -mtime +7 -delete

echo "备份完成: $DATE"
```

## 常见问题

### 1. 端口被占用

**问题**: 启动失败，提示端口被占用

**解决**: 
```bash
# 查看端口占用
lsof -i :8000

# 结束占用进程
kill -9 <PID>

# 或使用部署脚本自动切换端口
./deploy.sh
```

### 2. 镜像拉取失败

**问题**: 无法拉取Docker镜像

**解决**:
```bash
# 配置Docker镜像加速
# 编辑 /etc/docker/daemon.json
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://mirror.ccs.tencentyun.com"
  ]
}

# 重启Docker
sudo systemctl restart docker
```

### 3. 权限问题

**问题**: Docker命令需要sudo权限

**解决**:
```bash
# 将用户添加到docker组
sudo usermod -aG docker $USER

# 重新登录
logout
```

### 4. 磁盘空间不足

**问题**: Docker占用磁盘空间过大

**解决**:
```bash
# 清理未使用的资源
docker system prune -a

# 清理悬空镜像
docker image prune -a

# 清理构建缓存
docker builder prune -a
```

### 5. 容器无法启动

**问题**: 容器启动后立即退出

**解决**:
```bash
# 查看容器日志
docker compose logs backend

# 查看容器状态
docker compose ps

# 检查配置文件
docker compose config

# 重新构建
docker compose build --no-cache backend
docker compose up -d backend
```

### 6. 网络问题

**问题**: 容器间无法通信

**解决**:
```bash
# 检查网络
docker network ls
docker network inspect aistock_stock_network

# 重建网络
docker compose down
docker compose up -d
```

### 7. 数据库连接失败

**问题**: 后端无法连接MySQL

**解决**:
```bash
# 检查MySQL状态
docker compose ps mysql

# 查看MySQL日志
docker compose logs mysql

# 测试连接
docker compose exec backend python -c "from core.database import async_engine; print(async_engine.url)"
```

### 8. Selenium在ARM64上失败

**问题**: Selenium镜像不支持ARM64

**解决**:
```bash
# 部署脚本会自动使用seleniarm镜像
# 如果仍然失败，可以：
# 1. 使用远程Selenium服务
# 2. 使用云服务替代
# 3. 暂时跳过Selenium功能
```

      - --query_cache_size=0

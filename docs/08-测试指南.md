## 后端测试

### 环境准备

后端测试依赖已在 `requirements.txt` 中配置：

```bash
# 安装测试依赖
cd backend
pip install -r requirements.txt
```

### 运行后端测试

```bash
# 运行所有测试
pytest

# 运行单元测试
pytest tests/test_auth.py tests/test_rule_engine_unit.py

# 运行集成测试
pytest tests/test_data_sources.py tests/test_rules.py tests/test_tasks.py

# 运行回归测试
pytest tests/test_regression.py

# 运行黑盒测试（需要服务运行）
pytest tests/test_blackbox.py -m integration

# 生成覆盖率报告
pytest --cov=. --cov-report=html
```

### 测试文件结构

```
backend/tests/
├── conftest.py                    # pytest配置和fixtures
├── test_auth.py                   # 认证API集成测试
├── test_data_sources.py           # 数据源API集成测试
├── test_rules.py                  # 规则API集成测试
├── test_tasks.py                  # 任务API集成测试
├── test_rule_engine_unit.py       # 规则引擎单元测试
├── test_regression.py             # 回归测试套件
└── test_blackbox.py               # 黑盒/E2E测试
```

### 测试类型说明

#### 1. 单元测试
验证单个函数或方法的功能。

```bash
# 运行规则引擎单元测试
pytest tests/test_rule_engine_unit.py -v
```

#### 2. 集成测试
验证API端点的完整功能。

```bash
# 运行数据源API测试
pytest tests/test_data_sources.py -v
```

#### 3. 回归测试
确保新代码不会破坏现有功能。

```bash
# 运行回归测试
pytest tests/test_regression.py -v
```

#### 4. 黑盒测试
不依赖内部实现，通过HTTP接口进行测试。

```bash
# 确保服务正在运行
docker compose -f docker-compose.minimal.yml up -d backend

# 运行黑盒测试
pytest tests/test_blackbox.py -v -m integration
```

## 测试最佳实践

### 编写测试

1. **单元测试**
   - 每个测试只验证一个功能点
   - 使用清晰的测试名称
   - 遵循 AAA 模式（Arrange-Act-Assert）

2. **集成测试**
   - 使用 fixtures 管理测试数据
   - 确保测试相互独立
   - 清理测试后的数据

3. **回归测试**
   - 覆盖核心业务逻辑
   - 验证边界条件
   - 测试错误处理

### 测试命名

```javascript
// 良好的测试命名
describe('Login Component', () => {
  it('should display error message for invalid credentials', () => {
    // ...
  })
})
```

### 持续集成

建议在 CI/CD 流程中运行测试：

```yaml
# .github/workflows/test.yml 示例
name: Test Suite

on: [push, pull_request]

jobs:
  backend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run Backend Tests
        run: |
          cd backend
          pip install -r requirements.txt
          pytest --cov=. tests/

  frontend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run Frontend Tests
        run: |
          cd frontend
          npm install
          npm run test:unit
```

## 故障排除

### 后端测试常见问题

1. **数据库连接错误**
   - 确保测试使用独立的内存数据库
   - 检查 `conftest.py` 中的配置

2. **异步测试失败**
   - 使用 `@pytest.mark.asyncio` 装饰器
   - 确保 `pytest-asyncio` 已安装

### 前端测试常见问题

1. **Vue组件渲染失败**
   - 检查 `setup.js` 配置
   - 确保必要的全局组件已stub

2. **API模拟不工作**
   - 检查 `vi.mock()` 用法
   - 确保在导入前进行模拟

## 测试数据管理

### 使用Fixtures

```python
# 后端 - conftest.py
@pytest.fixture
async def test_user(db_session: AsyncSession) -> User:
    user = User(
        username="testuser",
        email="test@example.com",
        hashed_password=get_password_hash("testpass123")
    )
    db_session.add(user)
    await db_session.commit()
    return user
```

```javascript
// 前端 - setup.js
beforeEach(() => {
  localStorage.setItem('token', 'mock-token')
})
```

### 测试数据隔离

- 每个测试使用独立的测试数据
- 测试后清理数据
- 使用唯一标识符避免冲突

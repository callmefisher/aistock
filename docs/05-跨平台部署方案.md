# 跨平台自动适配部署方案

## 🎯 设计目标

1. **跨平台兼容**: 支持macOS (Intel/ARM64)、Windows、Linux (x86_64/ARM64)
2. **智能适配**: 自动检测操作系统和架构，选择合适的镜像
3. **高效部署**: 基础镜像只拉取一次，避免重复下载
4. **优化构建**: 多阶段构建，清理无效文件层
5. **端口自动检测**: 自动检测端口占用并切换

## 📋 支持平台

| 操作系统 | 架构 | Docker镜像 | 状态 |
|---------|------|-----------|------|
| macOS Intel | x86_64 | 标准镜像 | ✅ 完全支持 |
| macOS Apple Silicon | ARM64 | ARM64镜像 | ✅ 支持 |
| Windows | x86_64 | 标准镜像 | ✅ 完全支持 |
| Linux | x86_64 | 标准镜像 | ✅ 完全支持 |
| Linux | ARM64 | ARM64镜像 | ✅ 支持 |

## 🚀 快速开始

### 一键部署

```bash
# 1. 克隆项目
git clone git@github.com:callmefisher/aistock.git
cd aistock

# 2. 执行部署脚本
chmod +x deploy.sh
./deploy.sh
```

### 部署选项

```bash
# 正常部署
./deploy.sh

# 强制重新构建
./deploy.sh --force

# 不使用缓存构建
./deploy.sh --no-cache

# 查看帮助
./deploy.sh --help
```

## 🔧 核心特性

### 1. 自动平台检测

部署脚本会自动检测：
- 操作系统类型（macOS/Windows/Linux）
- CPU架构（x86_64/ARM64）
- 选择对应的Docker镜像

```bash
# 示例输出
[INFO] 检测平台: macos-arm64
[INFO] 检测到ARM64架构，使用seleniarm镜像
```

### 2. 智能镜像管理

#### 基础镜像策略
- **MySQL**: `mysql:8.0` (通用)
- **Redis**: `redis:7-alpine` (通用)
- **Nginx**: `nginx:alpine` (通用)
- **Selenium**: 
  - x86_64: `selenium/standalone-chrome:latest`
  - ARM64: `seleniarm/standalone-chromium:latest`

#### 镜像拉取逻辑
```bash
# 只在镜像不存在时才拉取
if ! docker image inspect mysql:8.0 &> /dev/null; then
    docker pull mysql:8.0
fi
```

### 3. 优化构建流程

#### 多阶段构建
```dockerfile
# 构建阶段
FROM python:3.11-slim as builder
# 安装依赖...

# 生产阶段
FROM python:3.11-slim as production
# 复制构建产物...
```

#### 清理无效层
```bash
# 清理apt缓存
rm -rf /var/lib/apt/lists/*

# 清理临时文件
rm -rf /tmp/* /var/tmp/*

# 清理Docker缓存
docker builder prune -f
```

### 4. 端口自动检测

自动检测端口占用并切换：

| 服务 | 默认端口 | 检测逻辑 |
|------|---------|---------|
| 前端 | 80 | 被占用则从8080开始查找 |
| 后端 | 8000 | 被占用则从8001开始查找 |
| MySQL | 3306 | 被占用则从3307开始查找 |
| Redis | 6379 | 被占用则从6380开始查找 |

## 📦 镜像优化

### 优化措施

1. **使用Alpine基础镜像**
   ```dockerfile
   FROM python:3.11-slim  # 而不是 python:3.11
   FROM nginx:alpine      # 而不是 nginx:latest
   FROM redis:7-alpine    # 而不是 redis:latest
   ```

2. **多阶段构建**
   - 构建阶段：安装编译依赖
   - 生产阶段：只包含运行时必需文件

3. **.dockerignore排除**
   ```
   __pycache__/
   .git/
   venv/
   *.log
   tests/
   docs/
   ```

4. **清理缓存**
   ```dockerfile
   RUN apt-get update && apt-get install -y \
       package-name \
       && rm -rf /var/lib/apt/lists/*
   ```

### 镜像大小对比

| 镜像 | 优化前 | 优化后 | 减少 |
|------|--------|--------|------|
| backend | ~800MB | ~400MB | 50% |
| mysql | ~550MB | ~550MB | - |
| redis | ~110MB | ~40MB | 64% |
| nginx | ~140MB | ~25MB | 82% |

## 🔄 部署流程

### 完整流程图

```
开始
  ↓
检测操作系统和架构
  ↓
检查Docker环境
  ↓
创建环境配置文件
  ↓
检测端口占用
  ├─ 端口可用 → 使用默认端口
  └─ 端口被占用 → 自动切换端口
  ↓
生成docker-compose.override.yml
  ↓
拉取基础镜像（仅首次）
  ├─ MySQL
  ├─ Redis
  ├─ Nginx
  └─ Selenium (根据架构选择)
  ↓
构建自定义镜像
  └─ backend (多阶段构建)
  ↓
启动服务
  ↓
显示访问地址
  ↓
完成
```

### 详细步骤

#### 1. 环境检测
```bash
check_docker()              # 检查Docker是否安装
check_docker_compose()      # 检查Docker Compose
detect_os()                 # 检测操作系统
detect_arch()               # 检测CPU架构
```

#### 2. 端口检测
```bash
check_port()                # 检查端口是否被占用
find_available_port()       # 查找可用端口
create_docker_compose_override()  # 生成覆盖配置
```

#### 3. 镜像管理
```bash
pull_image_if_needed()      # 按需拉取镜像
build_image_if_needed()     # 按需构建镜像
clean_docker_cache()        # 清理构建缓存
```

#### 4. 服务启动
```bash
docker compose up -d        # 启动所有服务
docker compose ps           # 检查服务状态
```

## 🔍 故障排查

### 常见问题

#### 1. 镜像拉取失败
```bash
# 配置镜像加速
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<EOF
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://mirror.ccs.tencentyun.com"
  ]
}
EOF

sudo systemctl restart docker
```

#### 2. 端口冲突
```bash
# 查看端口占用
lsof -i :8000

# 结束进程
kill -9 <PID>

# 或让脚本自动切换
./deploy.sh
```

#### 3. 权限问题
```bash
# Linux/Mac
sudo usermod -aG docker $USER
# 重新登录

# Windows
# 以管理员身份运行PowerShell
```

#### 4. 磁盘空间不足
```bash
# 清理Docker资源
docker system prune -a

# 清理构建缓存
docker builder prune -a
```

## 📊 性能优化

### 构建优化

1. **并行构建**
   ```bash
   docker compose build --parallel
   ```

2. **构建缓存**
   ```dockerfile
   # 先复制依赖文件
   COPY requirements.txt .
   RUN pip install -r requirements.txt
   
   # 再复制应用代码
   COPY . .
   ```

3. **层缓存**
   ```bash
   docker build --cache-from aistock-backend:latest .
   ```

### 运行优化

1. **资源限制**
   ```yaml
   services:
     backend:
       deploy:
         resources:
           limits:
             cpus: '2'
             memory: 2G
   ```

2. **健康检查**
   ```yaml
   healthcheck:
     test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
     interval: 30s
     timeout: 10s
     retries: 3
   ```

## 🔐 安全建议

1. **使用非root用户**
   ```dockerfile
   RUN useradd -m -u 1000 appuser
   USER appuser
   ```

2. **最小化基础镜像**
   ```dockerfile
   FROM python:3.11-slim  # 而不是 python:3.11
   ```

3. **不存储敏感信息**
   - 使用环境变量
   - 使用Docker secrets
   - 使用.env文件（不提交到Git）

4. **定期更新镜像**
   ```bash
   docker compose pull
   docker compose up -d
   ```

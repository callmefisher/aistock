## 核心模块设计

### 1. 登录处理模块 (LoginHandler)

#### 功能职责
- 支持多种登录方式：账号密码、验证码、二维码、Cookie
- 管理登录状态和Cookie持久化
- 处理登录失败和重试逻辑

#### 类设计
```python
class LoginHandler:
    def __init__(self, selenium_url, headless=True)
    def init_driver(self)                    # 初始化浏览器驱动
    def close_driver(self)                   # 关闭浏览器驱动
    def login_with_password(...)             # 账号密码登录
    def login_with_cookie(...)               # Cookie登录
    def handle_captcha(...)                  # 验证码处理
    def handle_qrcode_login(...)             # 二维码登录
    def save_cookies(...)                    # 保存Cookie
    def load_cookies(...)                    # 加载Cookie
```

#### 技术要点
- 使用Selenium WebDriver控制浏览器
- 支持远程Selenium服务（Docker环境）
- Cookie序列化和持久化存储
- 异常处理和重试机制

### 2. 数据抓取模块 (DataExtractor)

#### 功能职责
- 从网页表格提取数据
- 下载Excel文件
- 调用API接口获取数据
- 支持分页数据抓取

#### 类设计
```python
class DataExtractor:
    def extract_from_table(...)              # 从网页表格提取
    def download_excel(...)                  # 下载Excel文件
    def fetch_api_data(...)                  # API数据获取
    def extract_with_pagination(...)         # 分页数据提取
```

#### 技术要点
- 使用Pandas进行数据清洗和转换
- 支持多种数据格式（Excel、HTML、JSON）
- 处理动态加载的网页内容
- 支持Cookie认证访问

### 3. 规则引擎模块 (RuleEngine)

#### 功能职责
- 解析自然语言规则
- 转换为Excel公式和筛选条件
- 应用规则到数据集
- 验证规则有效性

#### 类设计
```python
class RuleEngine:
    def parse_natural_language(...)          # 解析自然语言
    def _parse_with_rules(...)               # 基于规则的解析
    def _generate_formula(...)               # 生成Excel公式
    def apply_rules(...)                     # 应用规则
    def validate_rules(...)                  # 验证规则
```

#### 技术要点
- 支持正则表达式匹配
- 可选集成OpenAI API增强解析能力
- 支持多种操作符（大于、小于、等于、包含等）
- 生成标准Excel公式

#### 支持的操作符
| 操作符 | 说明 | 示例 |
|--------|------|------|
| less_than | 小于 | PE < 20 |
| greater_than | 大于 | ROE > 15 |
| less_equal | 小于等于 | Price <= 100 |
| greater_equal | 大于等于 | Volume >= 1000 |
| equals | 等于 | Status = "active" |
| not_equals | 不等于 | Type != "ST" |
| contains | 包含 | Name contains "科技" |
| not_contains | 不包含 | Name not contains "退" |
| in | 在列表中 | Industry in ["科技", "医药"] |
| not_in | 不在列表中 | Type not in ["ST", "退市"] |

### 表结构说明

用户信息表，存储用户账户信息。

#### data_sources表
数据源配置表，存储网站登录信息和数据提取配置。

规则配置表，存储自然语言规则和转换后的Excel公式。

## API设计

### RESTful API规范

#### 命名规范
- 使用名词复数形式：`/api/v1/data-sources`
- 使用连字符分隔：`/api/v1/stock-pools`
- 避免动词：`GET /api/v1/tasks` 而不是 `GET /api/v1/get-tasks`

#### HTTP方法
| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 获取资源 | `GET /api/v1/tasks` |
| POST | 创建资源 | `POST /api/v1/tasks` |
| PUT | 更新资源 | `PUT /api/v1/tasks/1` |
| DELETE | 删除资源 | `DELETE /api/v1/tasks/1` |

#### 响应格式
```json
{
  "id": 1,
  "name": "任务名称",
  "status": "completed",
  "created_at": "2024-01-01T00:00:00"
}
```

#### 错误响应
```json
{
  "detail": "错误信息描述"
}
```

#### 认证相关

#### 数据源管理

#### 规则管理
- `GET /api/v1/rules` - 获取规则列表
- `POST /api/v1/rules` - 创建规则
- `GET /api/v1/rules/{id}` - 获取规则详情
- `PUT /api/v1/rules/{id}` - 更新规则
- `DELETE /api/v1/rules/{id}` - 删除规则
- `POST /api/v1/rules/{id}/validate` - 验证规则

#### 任务管理
- `GET /api/v1/tasks` - 获取任务列表

- `DELETE /api/v1/tasks/{id}` - 删除任务
- `POST /api/v1/tasks/{id}/run` - 执行任务

#### 选股池管理
- `GET /api/v1/stock-pools` - 获取选股池列表

## 安全设计

### 认证和授权
- **JWT认证**：使用JWT Token进行用户认证
- **密码加密**：使用bcrypt进行密码哈希
- **Token过期**：支持Token过期和刷新机制

### 数据安全
- **SQL注入防护**：使用SQLAlchemy ORM
- **XSS防护**：前端输入验证和转义
- **CSRF防护**：API使用Bearer Token认证

### 网络安全
- **HTTPS**：生产环境强制HTTPS
- **CORS**：配置跨域访问策略
- **Rate Limiting**：API访问频率限制（待实现）

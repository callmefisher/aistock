🎯 选股池自动化系统

技术方案对比与架构设计

📋 需求概要

  • 支持多种登录方式：账号密码、验证码、二维码、Cookie认证
  • 支持混合数据格式：Excel下载、网页表格、API接口
  • 智能规则引擎：自然语言转Excel公式/筛选
  • 多种输出形式：在线展示、Excel下载、邮件通知、历史记录
  • 灵活调度策略：支持多种定时任务配置
  • 本地Docker部署：一键启动，易于维护
  • 数据规模：约1万行，简化处理方案
🐍

方案一：Python全栈方案

FastAPI + Selenium + Pandas + OpenPyXL

核心技术栈

  • 后端框架：FastAPI（高性能异步框架）
  • Web自动化：Selenium（成熟稳定，支持各种登录）
  • Excel处理：Pandas + OpenPyXL（功能强大）
  • 任务调度：APScheduler（灵活的定时任务）
  • 前端界面：Vue3 + Element Plus（现代化UI）
  • 数据库：SQLite（轻量级，适合本地部署）
✅ 优势
  • Python生态成熟，数据处理能力强
  • Selenium支持复杂登录场景
  • Pandas处理Excel极其强大
  • 社区资源丰富，问题易解决
  • 学习曲线平缓，易于维护
❌ 劣势
  • Selenium需要浏览器驱动
  • 内存占用相对较高
  • 异步处理需要额外配置
🟢

方案二：Node.js现代化方案

Express + Puppeteer + ExcelJS + React

核心技术栈

  • 后端框架：Express.js（轻量级Web框架）
  • Web自动化：Puppeteer（现代化，性能好）
  • Excel处理：ExcelJS（功能完善的Excel库）
  • 任务调度：node-cron（Cron表达式调度）
  • 前端界面：React + Ant Design（企业级UI）
  • 数据库：SQLite（轻量级存储）
✅ 优势
  • Puppeteer性能优于Selenium
  • 异步处理天然优势
  • 内存占用相对较小
  • 现代化技术栈
❌ 劣势
  • Excel处理能力不如Pandas
  • 复杂规则转换实现较难
  • 需要Node.js生态知识
  • 社区资源相对较少
🚀

方案三：混合高性能方案

FastAPI + Playwright + Pandas + Vue3

核心技术栈

  • 后端框架：FastAPI（异步高性能）
  • Web自动化：Playwright（现代化，支持多浏览器）
  • Excel处理：Pandas + OpenPyXL（成熟方案）
  • 任务调度：Celery + Redis（分布式任务队列）
  • 前端界面：Vue3 + Naive UI（轻量级UI）
  • 数据库：PostgreSQL（功能完善）
✅ 优势
  • Playwright性能最佳
  • FastAPI异步性能强
  • Celery支持分布式任务
  • 整体架构可扩展性好
❌ 劣势
  • 架构复杂度高
  • 需要Redis等额外组件
  • 部署和维护成本高
  • 对于1万行数据过于重量级

🔄 核心功能流程

1. 数据源配置流程： 用户配置网站 → 测试登录 → 保存Cookie → 定期刷新认证 2. 数据抓取流程： 定时任务触发 → 加载网站配置 → 模拟登录 → 抓取数据 → 保存原始Excel 3. 规则处理流程： 用户输入规则 → NLP解析规则 → 转换Excel公式 → 应用到数据 → 生成选股池 4. 结果输出流程： 生成结果 → 在线展示 → Excel导出 → 邮件通知 → 保存历史记录 5. 自动化调度： 用户配置时间 → APScheduler调度 → 执行任务链 → 异常处理 → 结果通知

💡 推荐选择方案一：Python全栈方案

理由：开发效率高、Excel处理能力强、部署简单、维护成本低，完全满足1万行数据处理需求
